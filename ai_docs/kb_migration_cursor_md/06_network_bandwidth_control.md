# Network Bandwidth 제어 설계

## 목표

파일 병렬 복제 시 네트워크 대역폭을 과도하게 사용하지 않도록 제어한다.

## 필요성

Worker를 많이 실행하면 파일 복제 속도는 증가할 수 있지만, 네트워크 대역폭을 과도하게 점유할 수 있다.

따라서 전체 프로그램 단위 또는 Worker 단위로 전송 속도를 제한할 수 있어야 한다.

## 제어 단위

| 단위 | 설명 |
|---|---|
| Global Limit | 전체 프로그램의 총 전송 속도 제한 |
| Worker Limit | Worker별 전송 속도 제한 |
| Dynamic Limit | 운영 중 설정 변경 가능 |

## 대시보드 표시 항목

- 현재 Network 사용량
- 평균 Network 사용량
- 전체 전송량
- Worker별 전송량
- 초당 처리 파일 수
- 초당 처리 데이터 용량
- 설정된 Bandwidth Limit
- Limit 대비 사용률

## 구현 고려사항

- 파일 복제 Stream 처리 시 Throttling을 적용한다.
- 전체 전송량을 중앙에서 집계한다.
- Worker별 전송량도 별도로 수집한다.
- 제한값은 설정 파일 또는 UI에서 조정 가능하게 한다.
- 대용량 파일 복제 중에도 제한이 적용되어야 한다.

## Cursor 지침

```md
파일 복제 시 Network Bandwidth 사용량을 제어할 수 있는 구조를 설계한다.

전체 프로그램 단위의 전송 속도 제한과 Worker 단위의 전송 속도 제한을 고려한다.

대시보드에는 현재 전송 속도, 평균 전송 속도, 총 전송량, Worker별 전송량, 초당 처리 파일 수, 초당 처리 데이터 용량을 표시한다.

파일 복제 Stream 처리 단계에서 Throttling을 적용할 수 있도록 설계한다.
```
