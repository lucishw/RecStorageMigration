# 성능 검증용 테스트 데이터 생성 지침

## 목표

KB손보 파일 구조와 DB 데이터 샘플을 참고하여 검증용 테스트 데이터를 생성한다.

실제 운영 데이터를 직접 사용하지 않고, 패턴을 분석하여 무작위 테스트 데이터를 생성한다.

## 테스트 데이터 생성 대상

### 1. DB 대상 목록

- FileId
- SourcePath
- TargetPath
- FileSize
- FileExtension
- Status
- RetryCount
- CreatedAt

### 2. 파일 시스템 데이터

- 정상 파일
- 누락 파일
- 접근 불가 파일
- 중복 파일
- 대용량 파일
- 0 byte 파일
- 특수문자 포함 파일명
- 깊은 디렉터리 구조

## 패턴 분석 항목

- 파일 경로 패턴
- 파일명 패턴
- 파일 크기 분포
- 파일 확장자 유형
- 디렉터리 depth
- 일자 기반 경로 여부
- 업무 구분 코드 여부
- 고객/계약/문서 식별자 패턴 여부

## 테스트 데이터 규모 예시

| 규모 | 목적 |
|---|---|
| 1,000건 | 기본 기능 검증 |
| 100,000건 | 중간 규모 성능 검증 |
| 1,000,000건 | 대량 처리 성능 검증 |
| 10,000,000건 | 운영 규모 부하 검증 |

## Cursor 지침

```md
KB손보 파일 구조와 DB 샘플을 기반으로 성능 검증용 테스트 데이터를 생성할 수 있도록 한다.

실제 운영 데이터를 직접 사용하지 않고, 파일 경로 패턴, 파일명 패턴, 파일 크기 분포, 확장자 유형을 기반으로 무작위 테스트 데이터를 생성한다.

정상 파일뿐 아니라 누락 파일, 접근 불가 파일, 중복 파일, 대용량 파일, 0 byte 파일, 특수문자 포함 파일명도 생성하여 예외 상황을 검증할 수 있도록 한다.
```
