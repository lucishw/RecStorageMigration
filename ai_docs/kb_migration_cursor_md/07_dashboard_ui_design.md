# 대시보드 타입 UI 설계

## 목표

기존 단순 작업 화면이 아니라, 운영자와 엔지니어가 실시간으로 마이그레이션 현황을 파악할 수 있는 대시보드 형태로 UI를 설계한다.

## UI 설계 원칙

- 기존 프로그램 UI 스타일을 그대로 따르지 않는다.
- 운영자와 엔지니어가 실제로 필요한 정보를 우선 배치한다.
- 전체 상태를 한눈에 파악할 수 있어야 한다.
- 병목 지점을 빠르게 확인할 수 있어야 한다.
- 실패 파일과 오류 원인을 즉시 확인할 수 있어야 한다.
- Worker별 처리 상태를 실시간으로 확인할 수 있어야 한다.

## 화면 구성 예시

```text
┌─────────────────────────────────────────────────────────────┐
│ KB손보 파일 마이그레이션 대시보드                            │
├─────────────────────────────────────────────────────────────┤
│ 전체 대상: 10,000,000 | 완료: 4,500,000 | 실패: 1,200       │
│ 진행률: 45.0% | 현재 속도: 2,300 files/min | ETA: 03:12:20  │
├─────────────────────────────────────────────────────────────┤
│ Network: 420 Mbps / Limit 500 Mbps | CPU: 62% | MEM: 71%    │
├─────────────────────────────────────────────────────────────┤
│ Worker 현황                                                  │
│ Worker 1 | Processing | 350 files/min | current_file_001    │
│ Worker 2 | Processing | 340 files/min | current_file_002    │
│ Worker 3 | Error      | 0 files/min   | access denied       │
│ Worker N | Idle       | 0 files/min   | -                   │
├─────────────────────────────────────────────────────────────┤
│ 최근 오류                                                    │
│ FileId | SourcePath | ErrorCode | ErrorMessage | RetryCount │
└─────────────────────────────────────────────────────────────┘
```

## 전체 처리 현황

- 전체 대상 건수
- 처리 완료 건수
- 처리 실패 건수
- 대기 건수
- 진행률
- 현재 처리 속도
- 평균 처리 속도
- 예상 남은 시간
- 총 복제 용량
- 완료 복제 용량

## Worker 처리 현황

- Worker ID
- Worker 상태
- 현재 처리 중인 파일
- Worker별 완료 건수
- Worker별 실패 건수
- Worker별 처리 속도
- Worker별 전송량
- 마지막 처리 시각
- 마지막 오류 메시지

## 시스템 리소스 현황

- CPU 사용률
- Memory 사용률
- Disk I/O
- Network 사용량
- Network Bandwidth Limit
- DB Bulk Update Queue 크기
- Work Queue 크기

## 오류 현황

- 최근 오류 목록
- 오류 파일 경로
- 오류 코드
- 오류 메시지
- 재시도 횟수
- 실패 사유
- Worker ID
- 발생 시각

## Cursor 지침

```md
UI는 대시보드 타입으로 설계한다.

운영자 또는 엔지니어가 프로그램을 통해 운영 및 현황을 모니터링한다고 가정하고, 필요한 지표를 우선 배치한다.

대시보드에는 전체 진행률, Worker 단위 처리 현황, Network Bandwidth 사용 현황, 서버 리소스 현황, 오류 현황을 표시한다.

단순히 버튼과 로그만 있는 화면이 아니라 실시간 모니터링이 가능한 구조로 구현한다.
```
