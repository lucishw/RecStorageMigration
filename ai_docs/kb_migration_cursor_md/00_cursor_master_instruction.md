# Cursor 상위 지침

이 프로젝트는 KB손보 파일 마이그레이션을 위한 신규 .NET 8 기반 배치/대시보드 프로그램이다.

주요 목적은 DB 기준으로 대상 파일을 조회하고, 기존 스토리지의 파일을 TOBE 신규 스토리지로 복제하는 것이다.

기존 프로그램을 수정하는 방식이 아니라 신규 구조로 설계한다. 기존 UI나 기존 실행 방식에 얽매이지 않는다.

## 핵심 설계 방향

1. 하나의 프로그램 내부에서 N개의 Worker를 실행하여 병렬 파일 복제를 수행한다.
2. DB는 병목이 되지 않도록 Batch 조회와 Bulk Update 구조로 설계한다.
3. 파일 복제는 병렬 처리하되 Network Bandwidth 사용량을 제어할 수 있어야 한다.
4. UI는 기존 구조를 따르지 않고 운영자/엔지니어 관점의 대시보드 형태로 구현한다.
5. 대시보드에는 전체 진행률, Worker별 처리 현황, 서버 리소스, Network 사용량, 오류 현황을 실시간으로 표시한다.
6. KB손보 파일 구조와 DB 샘플을 기반으로 검증용 데이터를 생성하고, 기능 검증 및 성능 검증을 수행할 수 있어야 한다.
7. 전체 개발은 한 번에 진행하지 말고, 설계 → 구조화 → 기능 구현 → 검증 순서로 세부 Task를 나누어 진행한다.

## Cursor 작업 원칙

- 먼저 설계 문서를 작성한다.
- 설계 없이 바로 구현하지 않는다.
- 기존 프로그램 구조를 그대로 복사하지 않는다.
- 운영자와 엔지니어가 실제 운영 중 확인해야 하는 지표를 우선한다.
- DB 병목, 파일 I/O 병목, Network 병목을 분리해서 관찰할 수 있도록 설계한다.
- 모든 장기 실행 작업은 중단/재시작 가능성을 고려한다.
- 파일 단위 성공/실패 이력을 DB에 남긴다.
- 오류 발생 시 재처리 가능한 구조를 갖춘다.
